# LAN Clipboard Sync UI 设计

## 来源
本设计基于当前 Python CLI 版 LAN Clipboard Sync MVP，以及用户在视觉稿中选择的“A 和 C 的结合版”。这里的 A 指桌面控制台，C 指托盘小浮窗。

## 目标
为 LAN Clipboard Sync 增加一个可用的桌面 UI：主窗口用于配置、连接、状态查看和日志诊断；托盘/紧凑状态入口用于日常查看同步状态、暂停同步和重新打开主窗口。

## 范围
第一版 UI 聚焦 MVP 能力，不扩展图片、文件同步、云同步、设备自动发现、加密或历史记录。UI 应该驱动现有同步核心，而不是重写同步协议。

## 形态
采用“A + C 混合方案”：

- 主形态是桌面控制台，用户启动工具后可以看到完整状态。
- 辅助形态是托盘/紧凑状态入口，用于常驻后台后的快速操作。
- 第一版优先实现主窗口；托盘入口作为轻量能力实现，复杂菜单和系统级托盘兼容性可以后续增强。

## 主窗口布局
主窗口采用左侧导航和右侧内容区：

- 左侧：产品名、当前端口、导航项。
- 顶部：同步状态标题、开始同步、暂停、断开全部。
- 状态卡片：在线设备数、最近延迟、今日同步次数、循环拦截次数。
- 已连接设备：显示设备名、IP、在线/重连中状态。
- 手动连接：输入 IP、`host:port` 或 `ws://` 地址，并发起连接。
- 最近事件：显示服务启动、设备已连接、本地文本已同步、远端文本已应用、重复内容已拦截等日志。

## 托盘/紧凑入口
托盘/紧凑入口用于日常使用：

- 显示当前是否在线。
- 显示已连接设备数。
- 显示最近同步延迟。
- 显示监听端口。
- 提供暂停/恢复同步。
- 提供打开主窗口。

如果当前 Python/Tk 运行环境无法稳定提供真正系统托盘，第一版允许使用“最小化后保留小状态窗”的形式模拟托盘入口，但内部状态模型和按钮语义应与真正托盘一致。

## 交互流程
1. 用户打开 UI。
2. 用户输入本机端口和可选对端设备地址。
3. 用户点击“开始同步”。
4. UI 启动现有 `ClipboardSyncApp`。
5. 状态卡片和连接列表持续更新。
6. 用户可点击“暂停”停止 watcher/网络同步。
7. 用户可通过托盘/小浮窗查看状态并恢复主窗口。

## 技术方案
当前项目是无依赖 Python CLI。第一版 UI 使用 Python 标准库 `tkinter`，避免引入下载依赖。

建议新增模块：

- `lan_clipboard_sync/ui/state.py`：UI 状态模型和事件数据结构。
- `lan_clipboard_sync/ui/controller.py`：连接 UI 与现有同步核心，负责启动、暂停、停止和日志转发。
- `lan_clipboard_sync/ui/tk_app.py`：Tkinter 桌面主窗口和紧凑状态窗。
- `lan_clipboard_sync/ui/__main__.py`：UI 启动入口。

CLI 保持不变。新增运行方式：

```powershell
python -m lan_clipboard_sync.ui
```

## 视觉方向
界面应是工具型，而不是营销页：

- 信息密度适中，优先可扫读。
- 卡片半径不超过 8px。
- 主色使用清晰的连接蓝和在线绿，避免单一蓝/紫色主题。
- 日志区域使用等宽字体。
- 状态用颜色加文字，不只依赖颜色。
- 不使用装饰性渐变球、过大英雄区或说明性大段文案。

## 错误处理
UI 应明确显示：

- 端口占用。
- 设备地址格式错误。
- 剪贴板访问失败。
- 设备重连中。
- 同步核心启动失败。

错误信息应告诉用户发生了什么，以及下一步可以怎么处理。

## 测试策略
新增测试覆盖：

- UI 状态模型的默认值和状态转换。
- controller 对开始、暂停、停止的调用顺序。
- 设备地址输入规范化仍复用已有 `normalize_peer_url`。
- 日志事件能进入 UI 状态。

保留现有 16 个单元测试，并新增 UI 状态/controller 测试。Tkinter 视觉本身以手动运行和截图检查为主。

## 验收标准
- 用户可以通过 `python -m lan_clipboard_sync.ui` 打开桌面窗口。
- 用户可以在窗口里输入设备地址 / IP 并开始同步。
- 窗口能展示在线设备数、端口、运行状态和最近日志。
- 用户可以暂停/恢复同步。
- 用户可以打开紧凑状态窗或托盘模拟窗。
- CLI 入口 `python -m lan_clipboard_sync` 仍保持可用。
- 所有自动化测试通过。

## 非目标
- 不在第一版实现 Electron/Tauri。
- 不在第一版加入第三方 UI 库。
- 不在第一版做图片/文件同步。
- 不在第一版实现跨公网云同步。
- 不在第一版实现完整系统托盘兼容矩阵。
